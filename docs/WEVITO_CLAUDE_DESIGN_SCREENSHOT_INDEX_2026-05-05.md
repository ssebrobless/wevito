## Current Decision

Keep the existing overlay as the product center.

Claude Design is useful because it clarified tool surfaces, helper roles, and documentation shape. It is not a replacement for Wevito's living desktop-pet experience.

## Follow-Up Guidance

For code-side:

- use the screenshots as visual references only
- preserve overlay pass-through, pinned/unfocused behavior, and visible roaming pets
- keep PET TASKS report-only
- keep mutation-capable sprite work outside PET TASKS until explicitly coordinated

For visual-side:

- use these references when reviewing future UI screenshots
- prefer compact helper strips and status cards over broad dashboard takeover
- keep screenshot paths in future handoffs so other threads can inspect the same evidence
